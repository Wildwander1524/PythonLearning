① EAFP vs LBYL deeper

「请求原谅」比「请求许可」更容易 —— 为什么 Python 偏好 EAFP

LBYL ("Look Before You Leap") checks first, then acts. EAFP ("Easier to Ask Forgiveness than Permission") just acts, and handles the failure. Python prefers EAFP — and the sharpest reason is a bug LBYL can't see: the check can go stale before the use (a TOCTOU race).

LBYL 先检查再行动；EAFP 直接行动、失败再处理。Python 偏好 EAFP —— 最关键的原因是 LBYL 看不见的 bug：检查和使用之间，条件可能已经失效（TOCTOU 竞态）。

EAFP · idle

  • No race window. EAFP's lookup is one atomic step — nothing can slip between "check" and "use", because there is no separate check.
  • Less duplication / faster happy path. You don't state the condition twice; a try costs almost nothing until it actually raises.
  • Duck typing fit. EAFP asks "does it behave?" not "is it the right type?" — the Pythonic question.
  • When LBYL still wins: the action is expensive or irreversible and the check is cheap (validate before launching the rocket).
为什么偏好 EAFP：① 无竞态窗口（查找是原子的）；② 不重复条件、happy path 更快；③ 契合鸭子类型（问「能不能用」而非「是不是某类型」）。LBYL 仍合适的场景：动作昂贵/不可逆而检查便宜时——先验证，再发射火箭。

谁会运行、按什么顺序 —— 单步看 finally 永远运行
what does risky() do?
try except ValueError else finally after()
🪂 The amber rule. finally runs on every exit — success, a handled error, an unhandled error on its way out, even a return. Switch to KeyError (which the except ValueError can't catch) and watch: finally still fires, then the exception keeps propagating and after() is skipped.
🪂 琥珀色规则：finally 在所有退出路径上都运行——成功、已处理的异常、正在向外传播的未处理异常、甚至 return。切到 KeyError（except ValueError 抓不住）看：finally 照样运行，然后异常继续传播，after() 被跳过。

异常是继承树上的对象 —— except 匹配 = isinstance 检查

⑤ Modules, packages & __name__

模块 = 一个 .py；包 = 文件夹 + __init__.py；__name__ 守卫
how is logger.py reached?
  • 🍞 Import runs once, then caches in sys.modules: the first import executes the file top-to-bottom; later imports take a slice of the same loaf. Side effects fire once.
  • 🎭 __name__ is "__main__" only when the file is run directly; when imported it's the module's dotted name ("logkit.logger"). So the guard = "script mode only".
  • 📇 __init__.py is the package's front desk: from .logger import FileLogger lets callers write from logkit import FileLogger.
🍞 导入只运行一次并缓存（sys.modules），副作用只发生一次。🎭 直接运行时 __name__=="__main__"，被导入时是模块点名。📇 __init__.py 是包的前台，整理对外公开的名字。
