组合 vs 继承 —— 接口泄露：继承把父类的全部方法都暴露出去，组合让你只暴露想要的。

🔑 The rule, stated fairly. Inheritance for genuine is-a subtypes (a Manager is an Employee; framework bases like nn.Module). Composition for has-a and assembling behaviour (a Stack has a list; a Car has an engine). A stack is not a kind of list — it only wants to use one inside.
规则公正地说：继承用于真正的 is-a 子类型（Manager 是 Employee；框架基类如 nn.Module）；组合用于 has-a 与行为组装（Stack 有一个 list；Car 有一个 engine）。栈不是一种列表——它只想在内部使用一个列表。
