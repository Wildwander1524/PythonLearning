## 5. Docs-MCP tools compared + DeepWiki installed (folded from docs-mcp-tools-comparison)

**Context7 vs Docfork vs Deepcon vs DeepWiki — two categories:**
- **Library-docs servers:** Context7 (Upstash; ~50k libs, ~53.5k★, exposes Source-Reputation +
  primary URLs) · Docfork (9k libs, RAG+RRF, **"Cabinets"** stack version-pinning, MIT, ~463★) ·
  Deepcon (10k docs, "Query Composer", freemium ~$8–20/mo).
- **Repo-understanding:** **DeepWiki** (Cognition/Devin) — turns any public GitHub repo into a wiki +
  Q&A. *Complementary, not a Context7 competitor.*
- ⚠️ Rival "we beat Context7" benchmarks are **vendor self-reported**, not independent.

**Verdict:** keep **Context7 as the library-lesson default** (its provenance metadata is what powers
the citation discipline). **Docfork/Deepcon** add nothing for one-library-at-a-time lessons.
**DeepWiki is additive** → installed for the later repo-reading phase.

**DeepWiki install + verification:** remote MCP, **user scope**, `https://mcp.deepwiki.com/mcp`
(streamable HTTP). Tools: `read_wiki_structure`, `read_wiki_contents`, `ask_question`
(`owner/repo`, up to 10 repos). **Verified live** on `vllm-project/vllm` after restart:
`read_wiki_structure` → 13-section tree; `ask_question` → code-grounded `Scheduler` /
